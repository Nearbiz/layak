# LAYAK — Architecture Reference

## Stack decisions

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Frontend | React 18 + Vite + TypeScript + shadcn/ui | Fastest hackathon UI; component kit ready out of the box |
| API gateway | Go (Gin) — port 8080 | Single binary, fast deploys, easy on Fly.io |
| Score service | FastAPI (Python) — port 8001 | Score formula is numerical; Python is natural |
| Database | PostgreSQL 16 locally → OceanBase (Alibaba Cloud) | OceanBase is platinum sponsor; single bet |
| Credential signing | HMAC-SHA256 JWT | Proves the standard; implementable in 1 hour |
| Seed data format | JSON (workers.json + transactions.json) | Aein generates, Fiz ingests |
| Hosting | Fly.io or Railway | Single deploy command |

## Environment variables

| Var | Service | Description |
|-----|---------|-------------|
| `DATABASE_URL` | api, score | Postgres connection string |
| `API_PORT` | api | Default 8080 |
| `SCORE_SERVICE_URL` | api | URL to FastAPI score service |
| `JWT_SECRET` | api | HMAC-SHA256 signing key |
| `LAYAK_ISSUER` | api | `tng.layak.my` |
| `LENDER_KEY_AEON` | api | Demo shared secret for AEON Credit |
| `SCORE_PORT` | score | Default 8001 |
| `VITE_API_BASE_URL` | frontend | API base URL for browser |

## Local development quick-start

```bash
# 1. Start Postgres
make db-up

# 2. Copy env
cp .env.example .env

# 3. Run migrations
make migrate

# 4. Start services (3 terminals)
make api        # terminal 1
make score      # terminal 2
make frontend   # terminal 3

# 5. Open browser
# Worker:  http://localhost:5173/worker/worker_kumar_001
# Lender:  http://localhost:5173/lender
# TNG:     http://localhost:5173/tng
```

Or everything via Docker:
```bash
docker compose up --build
```

## OceanBase migration (post-hackathon)

OceanBase is MySQL-compatible. Steps when swapping:
1. Change `DATABASE_URL` DSN from `postgres://` to `mysql://`
2. Swap Go driver: `lib/pq` → `go-sql-driver/mysql`
3. Swap FastAPI driver: `psycopg2` → `aiomysql` or `sqlalchemy` with MySQL
4. Replace `JSONB` columns with `JSON` (OceanBase supports JSON type)
5. Replace `DECIMAL` precision syntax if needed
6. Re-run migrations against OceanBase instance

The SQL schema is intentionally minimal — the migration is ~1 hour of work.
